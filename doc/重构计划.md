# SinglePTZ-FaceTrack 重构计划

本文档记录 2026-04-16 在分支 `20260416codex` 上启动的重构方案。

目标不是继续在旧扫描逻辑上堆功能，而是把工程整理成适合调试、适合分阶段验证的结构，优先跑通以下 5 个阶段：

1. 单人静止
2. 单人移动
3. 4 个预置位巡检
4. 多人静止
5. 多人流动 + ReID

---

## 1. 现状判断

当前项目里真正值得保留的是底层能力，不是当前主流程。

可复用底层能力：

- RTSP 拉流
- Web MJPEG 预览
- YOLO 人体检测
- YOLO 人脸检测
- 海康 SDK 登录和 PTZ 基础调用
- 基础几何和日志工具

当前主流程的问题：

- `src/main.py` 启动时一次性加载检测、识别、图库、ReID、巡检状态机，耦合过重
- `src/scheduler/state_machine.py` 同时承担状态机、目标排序、PTZ 稳定判断、抓拍、识别、事件记录、Web 渲染
- 现有流程不是按“单人静止 -> 单人移动 -> 巡检 -> 多人 -> ReID”逐步构建出来的
- 当前业务逻辑偏向“巡检 + 放大 + 抓拍 + 识别”，不适合作为新主线起点

## 3. 新架构设计

新主线按以下模块拆分：

- `video_source`：视频流获取
- `detector`：人体/人脸检测
- `tracker`：目标跟踪
- `ptz_controller`：PTZ 控制
- `target_selector`：目标选择/排序
- `state_machine`：状态机
- `web`：视频查看和调试页面
- `scripts`：阶段脚本
- `hik_sdk`：海康 SDK 最小封装
- `common`：配置、日志、几何通用能力

## 7. 第一个阶段：单人静止的最小可运行方案

### 7.1 阶段目标

场景：

- 单人站在摄像头前，基本不动

目标：

1. 检测到人
2. 根据人体位置控制 PTZ 旋转
3. 逐步 zoom in
4. 检测到脸后切换为脸优先
5. 最终让脸居中并达到足够大小

### 7.2 阶段 1 需要的模块

- `video_source.rtsp`
- `detector.person_detector`
- `detector.face_detector`
- `tracker.single_target_tracker`
- `target_selector.single_target`
- `ptz_controller.hik_manual_controller`
- `web.server`
- `web.overlay`
- `state_machine.stage1_single_static`

### 7.3 阶段 1 最小状态机

建议状态：

1. `SEARCH_PERSON`
2. `CENTER_PERSON`
3. `ZOOM_STEP`
4. `CENTER_FACE`
5. `HOLD_FACE`
6. `LOST_TARGET`

### 7.4 阶段 1 控制策略

先跑通，再增强：

1. 用人体框做第一层闭环
2. 当脸出现且足够稳定时，切到脸闭环
3. PTZ 先用离散短脉冲控制，不先上复杂 PID
4. zoom 先做阶梯式放大，不直接做复杂 3D 定位
5. 脸丢失时退回人体，不直接丢目标

### 7.5 阶段 1 需要的 Web 调试信息

- 当前状态
- 当前目标类型：`person` / `face`
- 当前目标 bbox
- 目标中心误差 `dx/dy`
- 最近一次 PTZ 动作
- 当前是否处于 settle
- FPS
- 手动调试按钮：`left/right/up/down/zoom_in/zoom_out/stop/home/quit`

### 7.6 阶段 1 通过标准

建议以以下标准判断是否跑通：

1. 人进入画面后，摄像头会开始朝目标方向转动
2. 人体中心误差收敛到设定阈值内
3. 摄像头至少触发一次有效 zoom in
4. 检测到脸后，控制目标会从人体切换为脸
5. 最终进入 `HOLD_FACE`
