# Stage3 实现说明

当前版本对应 `stage3_preset_patrol` 四预置点巡航 + 基础追踪阶段。

目标：

- 无人时按 `1 -> 2 -> 3 -> 4 -> 1` 循环巡航
- 到达预置点后停留观察
- 观察到人脸后立刻进入基础追踪
- 追踪 10 秒后强制结束
- 结束后退回距离当前位置最近的全景预置点

## 当前入口

运行脚本：

- `scripts/stage3_preset_patrol.py`

核心代码：

- `src/stage3_preset_patrol.py`
- `src/ptz_position.py`
- `src/ptz_controller.py`

配置文件：

- `config/common.yaml`
- `config/stage3_preset_patrol.yaml`

## 当前状态机

当前状态与实现一致：

- `PATROLLING`
- `OBSERVING`
- `TRACKING`
- `RESETTING`

状态说明：

- `PATROLLING`
  - 依次调用预置点
  - 转动过程中不做检测
  - 通过 `patrol_settle_s` 等待画面稳定
- `OBSERVING`
  - 到达预置点后停留观察
  - 全景检测人脸和人体
  - 只有检测到人脸时才触发追踪
- `TRACKING`
  - 使用 Stage2 的核心思路做“傻瓜式”追踪
  - 人脸优先
  - 短时丢脸时允许回退人体
  - 目标居中且脸偏小时允许 `zoom_in`
- `RESETTING`
  - 追踪超时或丢目标后退出追踪
  - 读取当前绝对 PTZ
  - 计算距离哪个预置点最近
  - 直接调用最近预置点，退回全景

## 当前实现路线

1. 启动后登录海康 SDK
2. 读取 PTZ 范围和绝对坐标能力
3. 依次访问预置点 `1,2,3,4`，采样每个预置点的绝对 `Pan/Tilt/Zoom`
4. 启动正式巡航，从预置点 `1` 开始进入 `PATROLLING`
5. `PATROLLING` 结束后进入 `OBSERVING`
6. `OBSERVING` 内如果有人脸：
   - 进入 `TRACKING`
   - 启动 10 秒超时定时器
7. `TRACKING` 内使用：
   - 人体检测
   - 人脸检测
   - 单目标粘滞跟踪
   - 短时速度外推预测
   - 自适应 PTZ 速度和脉宽控制
8. `TRACKING` 超时或完全丢目标时：
   - 进入 `RESETTING`
   - 读取当前绝对 PTZ
   - 与 4 个预置点坐标做距离计算
   - 找最近预置点并回到该点
9. `RESETTING` 完成后重新进入 `OBSERVING`

## 当前与 Stage2 的关系

Stage3 没有复用 Stage2 的外层状态机，而是复用了其核心思想：

- 人脸优先、人体兜底
- 目标点预测
- 归一化中心误差
- 误差越大，云台速度和脉宽越大
- 目标居中后再决定是否放大
- 使用非阻塞 PTZ 控制，避免阻塞式 pulse 带来的明显卡顿

因此 Stage3 的 `TRACKING` 本质上是：

- 一个更简单的 Stage2 跟随子流程
- 但放在巡航状态机内部运行

## 当前 PTZ 绝对坐标能力

当前实现使用海康 SDK 读取：

- `NET_DVR_GET_PTZPOS = 293`
- `NET_DVR_GET_PTZSCOPE = 294`

读取结果用于两件事：

- 在启动阶段采样 4 个预置点的绝对 PTZ 坐标
- 在 `RESETTING` 阶段计算当前离哪个预置点最近

距离计算使用归一化欧氏距离：

- `pan` 使用环形距离
- `tilt` 使用线性距离
- `zoom` 使用线性距离

## 当前默认关键参数

来自 `config/stage3_preset_patrol.yaml`：

- `preset_ids: [1, 2, 3, 4]`
- `patrol_settle_s: 2.0`
- `observe_hold_s: 2.0`
- `tracking_timeout_s: 10.0`
- `reset_settle_s: 2.5`
- `preset_capture_settle_s: 2.0`
- `lost_face_to_body_s: 0.5`
- `lost_all_reset_s: 1.0`
- `desired_face_ratio_min: 0.055`
- `max_zoom_steps: 6`

这些参数对应当前阶段目标：

- 巡航简单稳定
- 检测与追踪逻辑清晰
- 不追求极限特写
- 追踪结束后优先快速退回全景

## 运行后应看到的效果

在当前实现下，预期现象是：

- 画面无人时，球机按 `1 -> 2 -> 3 -> 4 -> 1` 循环
- 到达预置点后静止观察
- 某个预置点出现人脸时，立刻进入追踪
- 追踪 10 秒后，自动结束并退回最近预置点

## 当前已知边界

当前阶段预期内存在这些边界：

- 如果同一个人一直站在某个预置点附近，回到该点后仍可能再次触发追踪
- `RESETTING` 目前是“最近预置点回退”，还没有去重记忆
- 追踪过程仍是基础闭环，不包含更复杂的局部搜索或多目标策略

这些问题属于下一阶段继续处理的内容，不属于 Stage3 当前实现的 bug。
